    Why is Portico using unusual terminology?

    One of Portico's goals with the new audit interface was to be more transparent. One way to be more transparent was to have the audit interface be a true window into the Portico archive, including the use of Portico internal terminology. We welcome your feedback on whether we should continue to use this terminology that matches the Portico content model, or instead use terminology that may be more familiar to our library and publisher auditors.

    What is a content type?

    The Portico archive is sliced into a limited set of content types. These content types allow us to group like content together and make it easier to manage both the content in the archive and the file system storage supporting the archive. As of December 2010, the content types within the Portico archive are:

    Content Type Description Available in the Audit Interface
    Metadata Files
    Business Artifacts Portico's license agreements with the publisher participants. Portico's archive is boot-strappable and self-describing, such that another organization could take the entirety of the archive and quickly begin working with the contents. To meet this requirement, the archive must contain the publisher agreements which allow Portico to archive the content.
    D-Collection Books Book content provided to Portico from publishers participating in the d-collection preservation service.
    D-Collection Newspapers Newspaper content provided to Portico from publishers participating in the d-collection preservation service.
    E-Book Content E-book content provided to Portico from publishers participating in the e-book preservation service.
    E-Journal Content E-journal content provided to Portico from publishers participating in the e-journal preservation service.
    Supplied Files for E-Book Content The original supplied files and their packaging for content provided as part of the e-book preservation service.
    Supplied Files for E-Journal Content The original supplied files and their packaging for content provided as part of the e-journal preservation service.

    What are the supplied categories?

    Portico transforms the publishers' original source files in two ways.

    1. The original XML file is transformed to the NLM archival format, and
    2. the original packaging is transformed to an archival format.

    As a backup, however, Portico preserves the original content exactly as it was provided by the publishers in the supplied files content types.

    What is the definition for the file formats?

    Images Images typically include page images, figure graphics, table graphics, and supplemental files (JPGs, GIFs, TIFs, etc.).
    Provider supplied text files The source XML files.
    Archive created text files The Portico created XML files, including both XML metadata or full-text files and the preservation metadata files.
    Application specific files Files meant to be opened within an application, such as MS Word or Excel.
    Executables Files that are programs or applications of their own.
    Multi-file packages Any of the package formats such as ZIP or TAR.
    Videos Video files.
    Audio Audio files.

    What is a provider?

    The provider (sometimes referred to as publisher), is the entity with which Portico has a signed license agreement allowing Portico to preserve content.

    Why is Portico a provider?

    Every item preserved in the archive must be associated with a provider. The license agreements and Portico documentation are associated with the Portico provider.

    Why is Public Domain a provider?

    Every item preserved in the archive much be associated with a provider. There are items such as openly available standards preserved in the archive and these are associated with the public domain provider.

    What is the difference between providers and publishers?

    In general, there is no difference. Provider is a nuanced term, however, as it is not unusual for any given item to have multiple entities that claim to be the publisher (e.g., a journal published by a society through another publisher) and thus we have used the term provider.

    What is this ingested date?

    The date on which that article, book, or other preserved item was ingested into the Portico archive.

    What is the Archival ID?

    Portico assigns unique IDs to individual preserved items, the files contained within those items, and even to elements of preservation metadata about the preserved items. Each of these unique identifiers is an archival identifier (ID).

    What are all those identifiers about?

    Providers often include multiple identifiers with each preserved item and Portico also assigns unique identifiers. Within the original XML file and the Portico archival, bibliographic metadata (or full-text) XML file, these identifiers are strongly typed. However, the Portico preservation metadata is used to drive the audit interface and within the preservation metadata is a simple Dublin Core metadata record which includes each and every identifier for the item. As one goal of the audit interface was to be more transparent, Portico is listing all these identifiers.

    What is Metadata Storage Info?

    This is information about the preservation metadata file that Portico creates for every preserved item.

    What is Content Unit Info

    Preserved items may have multiple versions (for example, perhaps the publisher provided Portico with a version of an e-journal article and then a year later provided an updated version). Each version is a different content unit and has a series of events associated with it.

    What is a content unit?

    Preserved items may have multiple versions (for example, perhaps the publisher provided Portico with a version of an e-journal article and then a year later provided an updated version). Each version is a different content unit and has a series of events associated with it.

    What are Events?

    Events are actions that Portico has taken on preserved items.

    What is the different between a Unit Event and a Content Unit Event?

    Events can occur on different elements of the preserved item. Conceptually, each preserved item is a single archival unit. That archival unit contains one or more content units. Unit Events are those events which are taken on the archival unit as a whole, whereas Content Unit Events are those events which Portico has taken on a specific content unit (version) of the preserved item.

    What is a D-Collection?

    As of December 2011, Portico provides libraries and publishers with three distinct preservation services: e-journals, e-books, and d-collections. A D-Collection contains content preserved via the d-collection preservation service. It is often (but not always) a digitized historical collection.
